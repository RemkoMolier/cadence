# Decisions

Architecture Decision Records (ADRs) capture the *why* layer of the corpus — the rationale behind significant choices.
Each ADR records one coherent decision: the context that motivated it, the alternatives considered, the choice made, and the consequences that follow.

## Adding a new ADR

1. Confirm the candidate is genuinely a separable decision and not a consequence of an existing ADR or a rule (which would belong in a standard).
1. Copy `template.md` to `NNNN-short-title-with-hyphens.md`, where `NNNN` is the next available four-digit number.
1. Fill in the frontmatter (`status: proposed`, `date`) and sections.
1. Open an integration request linked to the relevant work item.
1. On acceptance, change the frontmatter `status` from `proposed` to `accepted`.
